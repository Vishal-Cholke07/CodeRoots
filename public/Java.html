Introduction to Java

Java is a statically typed, object-oriented programming language known for its platform independence, security, and reliability. Its "write once, run anywhere" philosophy is made possible by the Java Virtual Machine (JVM).

  • Popular for web, Android, desktop, big data, scientific, and enterprise systems.
  • OpenJDK is the open-source reference implementation.
  • Runs on JVM so compiled bytecode can run on any supported OS.
  • Automatic garbage collection, strong standard library, huge ecosystem.
  • Used by Google, Netflix, banks, governments, and in millions of devices.

Installation & Setup

To start writing Java code, you need to install the JDK (Java Development Kit).

  • Download JDK from Adoptium (recommended), or Oracle.
  • Verify installation with java -version and javac -version in your terminal/command prompt.
  • Use any IDE: IntelliJ IDEA, Eclipse, NetBeans, or simple editors like VSCode.
  • Create and compile your first file:
// HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
  • To compile: javac HelloWorld.java
  • To run: java HelloWorld

Java Basic Syntax

Java code structure revolves around classes and methods. Every statement ends with ;. Files must be named after the public class they contain.

  • Public classes must match filename (Main.java has public class Main).
  • Main method: public static void main(String[] args)
  • Access modifiers: public, private, protected control visibility.
  • Use // for single-line and /* ... */ for multi-line comments.
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Type Conversion & Wrappers

Casting and wrapper classes allow conversion between types and between primitive types and classes (autoboxing).

  • Automatic conversion from small (byte, short) to int, int to double, etc.
  • Manual (explicit) casting needed for narrowing conversions.
  • Wrapper classes: Integer, Double, Boolean, Character, ...
  • Useful methods: Integer.parseInt(), Double.toString(), ...
double x = 5.5;
int y = (int) x; // y = 5
String s = String.valueOf(y);
Integer z = Integer.parseInt("42");

Control Flow in Java

Control structures direct execution. Includes if/else, switch (supports Strings and enums), and loop constructs.

  • if/else: any boolean condition
  • switch: works with int, char, String, enum
  • for, while, do..while: general, conditional, and post-tested loops
  • for-each loop: for (String s : list) {...}
  • break and continue modify loop behavior
int x = 10;
if (x > 5) {
    System.out.println("x is greater than 5");
}
for (int i = 0; i < 3; i++) {
    if (i == 2) continue;
    System.out.println(i);
}

Methods (Functions) in Java

Java methods live inside classes and define actions. They can be static or instance methods, can be overloaded, and may accept any number of parameters.

  • Methods have a return value or void.
  • Arguments can be primitive, object, or varargs sum(int... nums).
  • Recursive methods are supported.
public class Main {
    static int multiply(int a, int b) { return a * b; }
    static int sum(int... values) {
        int s = 0; for(int v: values) s += v;
        return s;
    }
    public static void main(String[] args) {
        System.out.println(multiply(5, 2)); // 10
        System.out.println(sum(1,2,3,4)); // 10
    }
}

OOP Principles in Java

Java is centered on OOP: organizing code into classes and objects. OOP makes software modular, reusable, secure, and maintainable.

  • Encapsulation: Hide fields, expose via methods (private fields + get/set)
  • Abstraction: Use abstract classes and interfaces to define behavior contracts
  • Inheritance: Code reuse via extends
  • Polymorphism: One interface, many implementations (method overriding, interface usage)
class Person { protected String name; }
class Student extends Person { int id; }
Person p = new Student();

Inheritance

Allows new classes to use (and extend) properties and behavior from a parent (base) class.

  • Use extends keyword for single inheritance.
  • Override methods using @Override annotation.
  • Use super to access parent methods/fields.
  • All classes ultimately extend Object.
class Animal { void speak() { System.out.println("Generic"); } }
class Dog extends Animal {
    @Override
    void speak() { System.out.println("Bark"); }
}
Animal d = new Dog(); d.speak(); // Bark

Interfaces & Abstraction

Interfaces allow multiple inheritance of type. Abstract classes share code but can't be directly instantiated.

  • Use interface keyword. Implement with implements; support default methods (Java 8+).
  • Abstract class: keyword abstract, may contain implementation.
  • A class can implement many interfaces, but extend only one parent class.
interface Drawable { void draw(); }
class Circle implements Drawable {
    public void draw() { System.out.println("Circle"); }
}
abstract class Shape { abstract void area(); }

Collections & Generics

Java Collections Framework provides dynamic containers (ArrayList, HashMap, HashSet, LinkedList, etc.). Generics ensure type-safety and clean code.

  • Use List, Map, Set for collections.
  • Enhanced for-loops for iterating.
  • Use Generics: ArrayList<String> names = new ArrayList<>();
  • Lambda and stream APIs simplify processing from Java 8+.
import java.util.*;
ArrayList nums = new ArrayList<>();
nums.add(2); nums.add(8);
for(int n: nums) System.out.println(n);
HashMap scores = new HashMap<>();
scores.put("Ram",90);
System.out.println(scores.get("Ram"));

Exception Handling

Java uses try-catch blocks to handle errors without crashing the program. There are checked and unchecked exceptions.

  • Use try-catch-finally to handle issues gracefully.
  • Throw exceptions with throw. Declare checked with throws.
  • Popular exceptions: NullPointerException, IOException, IllegalArgumentException.
  • Custom exceptions: extend Exception or RuntimeException.
try {
    int x = 5/0;
} catch (ArithmeticException e) {
    System.out.println("Error: " + e.getMessage());
} finally {
    System.out.println("Always runs");
}

File I/O

Java can read/write files using the java.io and java.nio libraries. Always handle IOExceptions.

  • Use FileReader, BufferedReader for reading; FileWriter, PrintWriter for writing.
  • Since Java 7+, use try-with-resources for auto-closing files.
import java.io.*;
try(BufferedReader br = new BufferedReader(new FileReader("test.txt"))) {
    String line;
    while((line = br.readLine()) != null) {
        System.out.println(line);
    }
} catch(IOException e) { e.printStackTrace(); }

Threads & Concurrency (Intro)

Java supports true multithreading—multiple tasks can run in parallel. Use Runnable or extend Thread class. Learn synchronized, Thread.sleep(), and concurrency utilities for advanced work.

  • Launch a thread: extend Thread or implement Runnable.
  • Coordinate access with synchronized or locks.
  • Use ExecutorService for advanced thread management (Java 5+).
class MyTask implements Runnable {
    public void run() {
        System.out.println("My thread running");
    }
}
Thread t = new Thread(new MyTask());
t.start();

Best Practices

  • Use meaningful class/method/variable names (camelCase).
  • Keep each class in its own file for large projects.
  • Avoid "magic numbers"; define as constants.
  • Minimize use of public fields; prefer private + getters/setters.
  • Document your code with comments and Javadoc.
  • Immutability (final fields/objects) is safer for multi-threading.
  • Avoid using null—prefer Optionals when possible (Java 8+).
  • Handle exceptions gracefully, close resources, never ignore errors.
