Introduction to C

C is a powerful, low-level programming language used for system and application software. Created in the early 1970s, it is the foundation for many modern languages like C++, Java, and Python. C is compiled, type-safe, and allows programmers direct control over memory.

  • C is a foundational language that influences C++, Java, Python, and more.
  • Widely used for operating systems, embedded devices, and performance-critical software.
  • Gives deep control over system resources (memory, CPU).
  • Comes with a standardized library (stdio, stdlib, string, math, etc.).
  • Encourages logical thinking and structured programming skills.

Environment Setup

To write and run C code, you need a C compiler like GCC, Clang, or software like Code::Blocks/Dev C++ (Windows) or you can use online IDEs.

  1. Install GCC:
    sudo apt update && sudo apt install build-essential (Linux)
    For Windows: Use MinGW-w64.
  2. Write code in any text editor, save with .c extension.
  3. Compile: gcc filename.c -o filename
  4. Run: ./filename or filename.exe (Windows)
  • C programs are written in text editors and saved with a .c extension.
  • Compilation translates your code to an executable file.
  • Errors and warnings are caught at compile time, not runtime.
  • Portable: C code can be run on many different hardware and OS, with slight tweaks.

Basic C Syntax

Each C program has one main() function. Statements end with semicolons. Code blocks are wrapped in curly braces.

#include <stdio.h>

int main() {
    printf("Hello, World!\n");
    return 0;
}
  • All statements end with a semicolon ;.
  • The main() function is the entry point of every C program.
  • Functions are declared before use, or a prototype is provided.
  • Curly braces {} define the start and end of code blocks.

Variables & Data Types

Variables are named memory locations. They must be declared with a type. Common types:

TypeKeywordExample
Integerintint age = 25;

  • C is statically typed: types must be declared.
  • Variables must be declared before use.
  • Variable names: can use letters, numbers, underscores but must not start with a digit.
  • Use const to declare constants, preventing accidental changes.

Conditional Statements

C uses if, else if, else, and switch for decision making.

int age = 20;
if (age >= 18) {
    printf("Adult\n");
} else {
    printf("Minor\n");
}
        
char grade = 'B';
switch (grade) {
    case 'A': printf("Excellent\n"); break;
    case 'B': printf("Good\n"); break;
    default: printf("Try again\n");
}
  • if: Executes a block if a condition is true.
  • else if: For multiple conditions.
  • else: Executes if none of the above are true.
  • switch: Best for checking a variable against multiple constant values.
  • You can use nested conditions for more complex logic.

Loops in C

Three types: for, while, and do...while loops.

for (int i = 0; i < 5; i++) {
    printf("%d ", i);
}
        
int n = 0;
while (n < 3) {
    printf("Loop %d\n", n);
    n++;
}
        
int m = 0;
do {
    printf("%d\n", m);
    m++;
} while (m < 2);
  • For loops: Best for known number of iterations.
  • While loops: Best when the condition may change within the loop.
  • Do-while loops: Always executes the loop body at least once.
  • Avoid infinite loops; make sure your condition eventually becomes false.

Arrays

Array is a collection of variables of the same data type.

int numbers[5] = {1, 2, 3, 4, 5};
printf("First: %d", numbers[0]);
  • Indexed from 0.
  • Size must be known at compile-time (static arrays) unless using malloc.
  • Useful for batch operations, function arguments, etc.

Strings

Strings in C are arrays of characters ending with a null character \0.

char greeting[] = "Hello";
printf("%s", greeting);
  • Strings are character arrays; terminated by null '\0'.
  • Common operations: read with scanf, print with printf, manipulate with strcpy, strlen (from string.h).
  • Remember to allocate space for the null character.

Functions

Functions are used to organize code. Each C program must have a main function.

int sum(int a, int b) {
    return a + b;
}

int main() {
    int result = sum(7, 5);
    printf("%d\n", result);
    return 0;
}

Functions can have parameters and a return type. Use void for no return value.

  • Breaks complex programs into smaller, manageable blocks.
  • Avoid code repetition by reusing functions.
  • Functions can return values or be void for no return.
  • Pass variables by value (default) or with pointers (for by-reference-like behavior).

Pointers

Pointers are variables that store memory addresses.

int x = 10;
int *p = &x;
printf("Value: %d, Address: %p", *p, p);

Pointers are essential for dynamic memory, arrays, and function arguments.

  • Pointers point to memory locations.
  • Necessary for dynamic memory, strings, and efficient function argument passing.
  • Syntax: int *p; (declares pointer), p = &var; (stores address).
  • Use with care! Incorrect usage can cause crashes or bugs.

Structs, Unions & Enums

Structs group different data types under a single name.

struct Student {
    int id;
    char name[20];
    float marks;
};

struct Student s1 = {1, "Alex", 82.5};
printf("ID: %d, Name: %s", s1.id, s1.name);

Enums assign names to integer constants:

enum Week {Sun, Mon, Tue, Wed, Thu, Fri, Sat};
enum Week today = Wed;
printf("Day: %d", today); // Outputs 3
  • Structs: Group multiple related variables of different types.
  • Unions: Similar to structs, but all members share the same memory (only one member can hold a value at once).
  • Enums: List of named integer constants; makes code readable.
  • Good for modeling real-world entities and organizing code.

File I/O

C provides file operations using fopen, fclose, fprintf, fscanf, etc.

FILE *fp = fopen("test.txt", "w");
if (fp != NULL) {
    fprintf(fp, "Hello, File!\n");
    fclose(fp);
}
        
// Reading from a file
char line[100];
FILE *fp = fopen("test.txt", "r");
if (fp != NULL) {
    fgets(line, sizeof(line), fp);
    printf("%s", line);
    fclose(fp);
}
  • Files must be opened with fopen() and closed with fclose().
  • fprintf and fscanf for formatted writing/reading.
  • fgets is safer than gets for reading strings.
  • Always check if file opening succeeded (pointer is not NULL).

Advanced Topics

Dynamic Memory Allocation uses malloc, calloc, realloc, and free from <stdlib.h>.

int *arr = (int*)malloc(5 * sizeof(int));
if (arr != NULL) {
    arr[0] = 10;
    free(arr);
}

Constants & Macros:

#define PI 3.14159
const int SIZE = 50;
printf("%f", PI);

Use header files to organize reusable code and custom functions.

  • Use malloc, calloc, free for dynamic memory.
  • Pointers and dynamic allocation make flexible data structures possible.
  • Macros (#define) let you write reusable symbolic constants or simple code logic.
  • Never forget to free memory to avoid leaks.
  • Custom header files (.h) help organize code.

Best Practices & Resources

  • Keep your code simple and readable; comment important logic.
  • Be mindful of array bounds and pointer arithmetic.
  • Regularly review code for memory leaks and logical errors.
  • Practice with small projects, experiment with all concepts above.
  • Use meaningful variable/function names.
  • Always initialize variables.
  • Free allocated memory to prevent leaks.
  • Use comments for clarity.
  • Practice by solving problems!
