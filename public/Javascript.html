Introduction to JavaScript

JavaScript (JS) is the programming language of the modern web, used for dynamic websites, apps, games, servers, mobile, embedded hardware, and more.

  • First created in 1995, now standardized as ECMAScript.
  • Mainly used in browsers, but also on servers (Node.js), and devices (IoT, React Native, Deno).
  • Dominates front-end development (with HTML, CSS).
  • Continuously growing, always evolving (ES6+ modern features).

Setup & Where JavaScript Runs

  • JavaScript runs natively in all browsers (Chrome, Firefox, Edge, Safari).
  • Node.js allows JS outside browsers (for backend, CLI tools, servers).
  • No install needed for simple web use—just open your browser’s console (F12 or Ctrl+Shift+J). 
    // In browser developer tools:
console.log('Hello from console');
  • Create HTML files with <script> tags, external JS files, or try playgrounds like JSFiddle, CodePen, Repl.it.
  • Node.js: node myapp.js runs JS on desktop/server.

Syntax & Variables

JavaScript syntax is flexible and forgiving. Code is written in statements, separated by semicolons (optional, but best practice), and organized in functions or files/scripts.

  • Variables can be declared with let, const (block scope, ES6+), or var (old, function/global scope).
  • Use let for values that change, const for constants, avoid var in new code.
  • Variable & function names: case-sensitive, camelCase, cannot start with number.
  • JS is dynamically typed: variables can hold any type at any time.
  • Hoisting: var declarations are moved up before code runs; let/const are not.
let age = 25;
const pi = 3.14;
var message = "Hello!";

console.log(typeof age); // number

Type Conversion & Coercion

Automatically or manually change between types.

  • Automatic conversion: let x = '5' * 2; // 10
  • Explicit: Number('42'), String(123), Boolean(0)
  • Falsy values: 0, '', null, undefined, NaN, false
  • parseInt, parseFloat convert from string
  • Always check edge cases. E.g., Number('abc') is NaN
Number('5') // 5
Boolean('') // false
String(99) // '99'
parseInt('42px') // 42

Control Flow

Use conditional code, multiple kinds of loops, and truthy/falsy logic.

  • if/else, switch for decisions
  • for, while, do...while, for...of, for...in for iteration
  • Break, continue, labeled blocks allowed
  • Ternary condition ? a : b
  • Optional chaining: user?.profile?.name (ES2020+)
for (let i = 1; i <= 3; i++) { console.log(i); }
let name = undefined;
console.log(name ?? 'Guest');  // Guest

Functions in JavaScript

First-class, flexible, and central to JS programming. Functions can be declared, as expressions, arrows, have default/rest/spread, and form closures for advanced programming.

  • Function declaration vs. function expression vs. arrow functions
  • Arguments object, default parameters, rest (...), spread usage
  • Closures: inner functions that remember outer scope
  • Recursion supported
  • IIFE (Immediately Invoked Function Expression)
function add(a, b=0) { return a + b; }
const square = x => x * x;
(function(){console.log('IIFE');})();
function outer() { let x=42; return function(){return x;} }

Arrays

Arrays are a core structure, highly dynamic, holding any type. Many powerful methods and patterns.

  • Common methods: push, pop, shift, unshift, slice, splice, map, filter, reduce, forEach, sort
  • Arrays can contain any type, mix types, and be nested
  • ES6: spread/rest, for...of, destructuring
let arr = [1, 2, 3];
arr.push(4);
let [first, ...rest] = arr;
console.log(arr.map(x => x * 2));

Objects

Objects in JS are dictionaries of key-value pairs and the foundation of most advanced structures. Properties/methods can be accessed and modified at runtime.

  • Create with curly braces or via constructors
  • Nested, dynamic property access (obj[key] or obj.key)
  • Object methods, this keyword
  • Spread and destructuring (ES6+)
  • Prototype chain: inheritance via __proto__ or Object.create()
const user = { name: "Priya", age: 20 };
console.log(user["name"]);
const {age} = user;  // destructuring

OOP: Classes & Prototypes

JavaScript supports OOP via prototypes and since ES6+ with class syntax. Real OOP patterns: inheritance, encapsulation, polymorphism.

  • Pre-ES6: Constructor functions, prototype property
  • ES6+: class, constructor, extends, super, static fields, private fields (#foo)
  • Polymorphism possible via method overriding
class Car {
  #internalID = 123;
  constructor(brand, year) {
    this.brand = brand;
    this.year = year;
  }
  drive() { console.log("Driving..."); }
}
class ElectricCar extends Car {
  drive() { console.log("E-driving..."); }
}
const myCar = new ElectricCar('Tesla', 2021);
myCar.drive();

Async JavaScript

Many JS operations (AJAX, APIs, timers, UI) are asynchronous. JS handles async via callbacks, promises, async/await—critical for modern web programming.

  • Callbacks: pass a function for later execution
  • Promises: object representing future completion/failure (MDN Promise)
  • Async/await: elegant modern syntax (ES2017+)
  • AJAX: Asynchronous HTTP with fetch() or XMLHttpRequest
  • setTimeout/setInterval (timers)
setTimeout(() => console.log('After 1s!'), 1000);
fetch('https://jsonplaceholder.typicode.com/todos/1')
  .then(res => res.json())
  .then(data => console.log(data));
async function foo() {
  let res = await fetch('https://api.example.com/data');
  return res.text();
}

DOM & Browser APIs

JS manipulates HTML and CSS via the DOM (Document Object Model). Numerous powerful APIs are available for web interactivity.

  • Querying: document.getElementById, querySelector
  • Changing content/styles dynamically
  • Events: onclick, addEventListener, keyboard/mouse events
  • APIs: localStorage, fetch, navigator, history, Canvas, geolocation
document.querySelector('#btn').addEventListener('click', () => {
  alert('You clicked!');
});
localStorage.setItem('theme', 'dark');

Error Handling

JS uses try/catch/finally to handle errors—critical for robustness, especially in async or UI code.

  • Native errors: Error, TypeError, ReferenceError, etc.
  • Custom errors: extend Error
  • Throw, catch, recover as needed
try {
  throw new Error('Oops!');
} catch(e) {
  console.log(e.message);
} finally {
  console.log('Always runs');
}

Modern JS (ES6+)

Recent specs bring new syntax and power, including arrow functions, template literals, destructuring, modules, let/const, defaults, async/await, and more.

  • Use let and const (not var)
  • Import/export modules (import x from './x.js')
  • Template strings: const msg = `Hi, ${user}!`;
  • Destructuring: const {id, name} = obj;
  • ...spread operator, REST (...args), default parameters
const user = {name: "Aman", id: 10};
const {id, ...rest} = user;
import add from './math.js';

Popular Libraries & Tools

  • npm, yarn: package managers for libraries (npm)
  • React, Vue, Svelte: front-end frameworks
  • Node.js: server-side JS, CLI tooling
  • jQuery: classic DOM utility (often unnecessary now)
  • TypeScript: superset of JS adding static typing (TS)
  • Lodash/Underscore: common utilities
  • Webpack/Parcel: module bundlers
  • ESLint/Prettier: code quality, formatting

Best Practices

  • Prefer let/const over var
  • Name variables in camelCase, constants in SCREAMING_SNAKE_CASE
  • Keep code DRY (Don’t Repeat Yourself)
  • Avoid globals, keep functions short
  • Write modular code (use modules)
  • Comment your code, use JSdoc for documentation
  • Sanitize/validate user input (security)
  • Handle errors, always test code in multiple browsers
