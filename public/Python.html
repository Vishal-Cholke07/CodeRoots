Introduction to Python

Python is a top open-source programming language, known for its clarity and enormous ecosystem. Created by Guido van Rossum in 1991. Used everywhere: web, data science, automation, AI, games, desktop, and more.

  • High-level and easy to read—great for beginners and pros.
  • Dynamically typed, interpreted (no compile step required).
  • Huge library support (standard + PyPI packages).
  • Widely used in Google, NASA, Spotify, Netflix, YouTube, academia.

Setup & Installation

  • Download from python.org (choose latest 3.x version).
  • Recommended IDEs: PyCharm, VSCode, Thonny, or simple editors.
  • Run code: terminal (python script.py), IDLE, or online (Replit, Programiz).
  • REPL mode: run python in terminal for interactive prompt.
  • pip: Python package manager for third-party modules.
$ python --version
$ pip install pandas

Syntax & Indentation

  • Uses colons (:) and indentation to mark blocks (not braces).
  • Consistent 4-space indent standard (see PEP8).
  • Comments: # single line, '''multi line'''.
  • No end-statement character; end of line means end of statement.
# This is a comment
def greet(name):
    print(f"Hello, {name}")

Type Conversion

  • Automatic conversion when possible, else explicit cast: int(), float(), str(), list()
  • Convert string to number: int('57'), float('3.2')
  • Convert list to set: set([1,2,3,3]) → {1,2,3}
x = '42'
print(int(x) + 1)  # 43
nums = list('abc') # ['a', 'b', 'c']

Control Flow

  • if, elif, else conditions
  • while and for loops (iterate lists, dicts, ranges, etc.)
  • 'break', 'continue', 'pass' for flow control
  • Python 3.10+: match/case structural pattern matching
for i in range(3):
    if i == 1: continue
    print(i)

value = 2
match value:
    case 1: print("one")
    case 2: print("two")
    case _: print("other")

Functions

  • Define with def
  • Supports default, keyword, variable-length (*args, **kwargs) arguments
  • Lambda functions: lambda x: x+1
  • Docstrings for documentation
def factorial(n):
    """Recursively calculate n!"""
    return 1 if n <= 1 else n * factorial(n-1)
print(factorial(5))  # 120

Collections / Sequences

  • list: ordered, mutable
  • tuple: ordered, immutable
  • set: unique unordered items
  • dict: key-value pairs (order guaranteed since Python 3.7)
lst = [1, 2, 2, 3]
unique = set(lst)
d = {'a': 1, 'b': 2}
print(list(d.keys()))

Strings

  • Immutability: cannot change in-place
  • f-strings (formatted): f"Hello {name}" (Python 3.6+)
  • Common methods: .split(), .join(), .upper(), .replace()
  • Multi-line (triple quotes) and raw strings (r'\t')
s = "Hello, World!"
print(s.lower())
print(f"Len: {len(s)}")

List, Dict, Set Comprehensions

  • Powerful, compact syntax for generating lists, dicts, sets
  • List: [x*x for x in range(5)]
  • Dict: {k: v*2 for k, v in d.items()}
  • Set: {x for x in lst if x > 0}
evens = [x for x in range(10) if x%2==0]
sq_dict = {x: x*x for x in range(4)}

Object-Oriented Programming (OOP)

  • Classes: class X: with __init__ method for initialization
  • Single & multiple inheritance with explicit parent list
  • Instance vs class attributes; self always first argument
  • Special methods: __str__, __repr__, __len__, __getitem__
  • @staticmethod and @classmethod supported
  • Dunder for operator overloading
class Animal:
    def speak(self):
        print("Generic sound")
class Cat(Animal):
    def speak(self):
        print("Meow")

c = Cat()
c.speak()

Exception Handling

  • try/except/else/finally
  • Raise exceptions: raise ValueError('No!')
  • Custom exceptions: subclass Exception
  • All errors inherit from BaseException
try:
    n = int(input())
except ValueError:
    print('Invalid input')
finally:
    print('Done')

File I/O

  • Open files with with open('file.txt') as f
  • Read/write lines with .read(), .write(), .readlines()
  • Always close files, or use a context manager (with)
with open('test.txt', 'w') as f:
    f.write('Hello')
with open('test.txt') as f:
    line = f.readline()

Modules & Packages

  • Import with import, from ... import...
  • Standard library: os, sys, math, random, datetime, re, json, statistics
  • Install packages via pip, use venv for environments
import math, random
print(math.pi, random.randint(1,10))

Decorators & Higher-Order Functions

  • Functions can accept and return other functions.
  • Decorators modify function/class behavior (@decorator syntax)
def bold(func):
    def wrapper():
        return '' + func() + ''
    return wrapper

@bold
def greet():
    return "Hello"

Generators & Iterators

  • Generator functions use yield for memory-efficient iteration
  • Use next() to get values one at a time
  • All containers are iterable; customize iteration with __iter__
def countdown(n):
    while n > 0:
        yield n
        n -= 1
for i in countdown(3):
    print(i)

Best Practices

  • Follow PEP 8 coding standards (naming, whitespace)
  • Use venv for dependencies
  • Comment, use docstrings, and type hints (Python 3.5+)
  • Prefer clear, readable code over cleverness
  • Write unit tests (e.g. with unittest, pytest)
  • Don’t reinvent the wheel—use the standard library
  • Be explicit: is vs. ==, None checks, exceptions
