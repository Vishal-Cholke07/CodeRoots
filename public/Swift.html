Introduction to Swift

Swift is Apple’s open-source, type-safe, and expressive language for iOS, macOS, watchOS, and tvOS. It is designed for speed and safety, blending the power of C and Objective-C with modern language features. Swift is fun and efficient, and it’s the best way to build Apple platform software today.

  • Modern, concise syntax designed for readability and performance.
  • Memory safe: prevents null pointer bugs with optionals, enforces value initialization.
  • Used worldwide for iOS apps, Apple Watch, Mac desktops – with reach to open source.
  • Supported by a vibrant developer community and constantly improved by Apple.
  • Favorite for both prototyping and enterprise-grade apps.

Setup & Tools

Swift development is easiest with Apple’s Xcode IDE (macOS). Windows/Linux users can build and play with Swift via the Swift.org toolchain. Learning is fun with Swift Playgrounds (macOS, iPad), or online playgrounds.

  • Xcode is free and preferred for app dev: syntax highlighting, playgrounds, simulators, UI design.
  • swiftfiddle.com and online.swiftplayground.run allow quick code tests.
  • Install Swift CLI from Swift.org for command-line/VSCode workflow.
  • Write scripts in .swift files. Run swift myfile.swift or swift repl for interactive usage.
// Terminal
swift --version
// Xcode Playground
print("Hello from Swift Playground!")

Syntax & Variables

Swift’s syntax focuses on brevity, clarity, and type safety. All variables/constants must be explicitly declared. Swift makes heavy use of inference, so types need not always be spelled out. Immutability is encouraged.

  • let declares an immutable constant, var declares a mutable variable.
  • Semicolons are OPTIONAL & barely used; whitespace/indentation is meaningful.
  • String interpolation: "Hello, \(name)!"
  • Single line comments //, multi-line /* ... */
  • Identifiers (names) are Unicode-safe (emojis allowed!)
let language = "Swift" // constant
var counter: Int = 0   // mutable variable
var greeting = "Hello, world!" // inference

Data Types & Optionals

Every variable’s type must be clear. Swift includes powerful primitives, rich composite types, and optionals for safety. Types can be explicit or inferred, and the type system rejects inconsistencies at compile time.

  • Basic types: Int, Double, Float, Bool, String, Character.
  • Type annotations (after :) are optional if value is obvious.
  • Collections: Array, Set, Dictionary.
  • Optionals: A variable that may/may not have a value. var age: Int?
  • nil: Absence of value (not the same as 0 or empty string)
  • All types are structs except classes.
let pi: Double = 3.14159
var name: String = "Alex"
var optionalNumber: Int? = nil
var tags: [String] = []
var user: [String: String] = [:]

Type Inference & Conversion

Swift automatically infers the type if the assignment is clear. You can convert compatible types with initializers.

  • Type annotation (: Type) only needed for clarity, not required if obvious.
  • Convert String to Int: Int("42") (returns optional Int?)
  • Convert Int to String: String(42)
  • Be careful: forced unwrapping fails if conversion is invalid!
let strNumber = "5"
if let n = Int(strNumber) {
  print(n * 2) // 10
}

Operators

  • Arithmetic: + - * / %
  • Assignment: = += -= *= /=
  • Comparison: < > <= >= == !=
  • Logical: && || !
  • Range: 0...5 (inclusive); 0..<5 (exclusive)
  • Nil-coalescing: a ?? b (returns a if not nil, else b)
  • Identity: === (is identical object, for classes)
let a = 7, b = 3
print(a % b)
let input: String? = nil
print(input ?? "No value")

Control Flow

Swift improves classic C-family flow with pattern matching and safer constructs. Conditions must be booleans (not like JS/Python truthy/falsy).

  • if, else if, else
  • switch (powerful: supports patterns, ranges, tuples, enums)
  • for-in loop, while, repeat-while
  • Use guard for early exits and safer code
  • Optional binding/unwrapping: if let, guard let
  • break, continue supported
for n in 1...3 {
  print(n)
}
let name: String? = nil
print(name ?? "Guest")

Functions

Swift’s functions are powerful: support default params, inout (ref) params, variadic args, nested/local functions, closures, multiple return types (tuples).

  • Define with func; can omit return type if void.
  • Argument labels (external and internal) for clarity
  • Return tuple for multiple values
  • Swift closures (lambdas) are ubiquitous – e.g. sorted, map, callbacks
func greet(name: String = "Guest") {
  print("Hello, \(name)")
}
greet()

Collections

Swift has three main collections: Array, Set, Dictionary, all value types (structs) and generics.

  • Array: ordered, duplicates allowed
  • Set: unordered, unique values only
  • Dictionary: key-value mapping, fast lookup
  • For-in loop to iterate all; many functional methods: map, filter, reduce
var nums = [1, 2, 3]
nums.append(4)
var names: Set = ["Amy", "Bob", "Amy"]
let capitals = ["FR": "Paris", "UK": "London"]

Optionals

Optionals represent a variable that could be nil or have a value—this is Swift's safety net against null pointer bugs.

  • Declare with ?: var x: Int?
  • Must be unwrapped before used (with if let, guard let, !, or ??)
  • Chaining: user?.profile?.email
  • Fast, safe pattern for many real-world APIs and data flows.
var optional: Int? = nil
if let value = optional {
  print(value)
} else {
  print("No value!")
}

Structs vs Classes

Swift allows both struct and class types, but structs are value types (copied) and classes are reference types (shared reference). Most types in Swift stdlib are structs!

  • Structs: value semantics, lightweight, no inheritance
  • Classes: reference semantics, support inheritance, deinitializers
  • Both: can have methods, computed properties, initializers
  • Protocol conformance, extensions work for both
struct Point { var x=0, y=0 }
class Shape { var sides=0; init(sides:Int){ self.sides=sides } }

Swift OOP

Swift brings modern OOP and protocol-oriented programming:

  • Inheritance: class-based, use : Parent
  • Protocols: define methods/props requirement like interfaces, and can add default behaviors in extensions
  • Extensions: add methods to any type (even stdlib) without subclassing
  • Initializers: many options (designated/convenience, required, etc.)
  • Enums: very powerful—can have methods, associated values, conform to protocols
  • Static/class methods for type-level functionality
  • Operator overloading possible on value types
protocol Drawable { func draw() }
extension Drawable {
  func showDemo() { print("Demo") }
}
class Circle: Drawable {
  func draw() { print("O") }
}
let c = Circle(); c.draw(); c.showDemo()

Error Handling

  • Swift uses do/try/catch (and throw) for error management.
  • Functions/methods that can fail must be marked throws
  • try? returns optional (nil if error thrown), try! asserts no error will happen (unsafe!)
  • Error enums: define specific failure cases
enum FileError: Error { case missing, unreadable }
func open(file: String) throws {
  if file.isEmpty { throw FileError.missing }
}
do {
  try open(file: "")
} catch {
  print("Error: \(error)")
}

Modern Swift Patterns

  • map, filter, reduce, forEach, flatMap, compactMap on collections
  • Closures (lambda syntax): { (args) -> Return in ... }; trailing closure syntax
  • Protocol-oriented, extension-oriented design: key to idiomatic Swift
  • Generics for reusable code (e.g. func swap)
  • Property observers, computed properties, subscripts
let numbers = [1,2,3,4].filter { $0 % 2 == 0 }

Best Practices

  • Always use let when you don’t need mutation
  • Prefer structs and value semantics for custom types for simple data models
  • Write concise, readable code with clear names
  • Take advantage of optionals, extensions, and protocols
  • Use guard/if-let for safe unwrapping
  • Document your code (Xcode’s Quick Help = /// comments)
  • Rely on SwiftLint (SwiftLint) for style, never force unwrap unless certain
  • Test using XCTest; use Xcode for debugging
